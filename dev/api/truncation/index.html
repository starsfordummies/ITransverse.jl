Truncation & Sweeps

Algorithms for compressing an MPS resulting from an MPO-MPS product.

Core contraction / application

ITransverse.tcontract — Function

Copied from ITensorMPS's contract but adapted so that it can also extend. Extension is done at the right (top), ie. contraction goes like 

    | | | | | | |
A = o-o-o-o-o-o-o 
    | | | | |  
psi o o o o o
source

Contract MPO-MPS with algorithm densitymatrix, starting from the left. At the end we can chop/extend if we work with light cone. If contract_dangling=true we shrink dangling edges, if present

ITransverse.tapplys — Function

Applies MPO to MPS from the other side (p' MPO leg, leaving open the p leg): instead of

psi--(p)-(p)--[O]--(p')-- = Opsi--(p)--

it contracts

--(p)--[O]--(p')-(p)--psi = -(p)--Opsi

RTM-based sweeps

ITransverse.truncate_sweep — Function

Our original algorithm: first bring to canonical form, then perform gauge transformations and truncations. Could call it "naiveRTM"

source
ITransverse.truncate_sweep_sym — Function

Symmetric truncate for MPS optimizing the RTM |psi*><psi|. direction = :left → sweeps 1→N direction = :right → sweeps N→1

source

truncate_sweep_rtm, truncate_lsweep_sym, truncate_rsweep_sym are additional sweep variants (currently undocumented).

Left-right truncation

ITransverse.tlrapply — Function

Applies AL to ψL from the left, AR to ψR from the right, and truncates on the RTM |ARR><LAL| by building it explicitly.
Convention: when direction=:left, we PTR over left environments and, going right→left, we SVD τR = trL(τ).
Returns TruncLR (destructures as (LEFT, RIGHT, SV))

source
ITransverse.trapply — Function

Applies MPO to the right and truncates on the RTM |AR*R><L| by building it explicitly
Returns TruncLR (destructures as (LEFT, RIGHT, SV))

source
ITransverse.tlapply — Function

Applies MPO to the left and truncates on the RTM |R><L*AL| by building it explicitly
Returns TruncLR (destructures as (LEFT, RIGHT, SV))

source
